# Obsidian Vault Migration — Design Spec

**Date:** 2026-04-25
**Branch:** `feat/obsidian-vault-migration`
**Scope:** Migrate codebase-notes plugin from flat-file `~/.claude/repo_notes/` storage to Obsidian vault-based storage at `~/vaults/<repo-slug>/`.

---

## 1. Overview

Two deliverables on a single feature branch:

1. **Migration script** — one-off converter from `~/.claude/repo_notes/<repo_id>/` to `~/vaults/<slug>/`
2. **Plugin rework** — all skills, scripts, and hooks updated to create and manage Obsidian vaults natively

### Design Principles

- Wikilinks everywhere (replace all relative markdown links)
- Obsidian Excalidraw plugin replaces PNG rendering pipeline
- Hot cache pattern (from claude-obsidian) replaces context index injection
- Keep staleness detection, code review, and commit scripts — adapt paths
- Drop nav_links.py, render.py, context_index.py
- One vault per repo, stored at `~/vaults/<repo-slug>/`

---

## 2. Vault Structure

Each repo gets a vault at `~/vaults/<repo-slug>/`:

```
~/vaults/<repo-slug>/
├── .obsidian/                    # Obsidian config
│   ├── app.json                  # UI defaults
│   ├── appearance.json           # Theme
│   ├── core-plugins.json         # Enable backlinks, graph, properties, outgoing-links
│   ├── community-plugins.json    # Excalidraw, Dataview, Templater
│   ├── graph.json                # Color groups by folder
│   └── snippets/
│       └── vault-colors.css      # Folder-based color scheme
├── notes/                        # Codebase knowledge
│   ├── overview.md               # Root knowledge map (was 00-overview.md)
│   ├── <topic>/                  # No NN- prefix
│   │   ├── index.md
│   │   ├── <subtopic>.md         # No NN- prefix
│   │   └── <subtopic>.excalidraw # No .png — Excalidraw plugin renders
│   └── ...
├── research/                     # External papers, articles
├── projects/                     # Brainstorming, planning
├── commits/                      # Git history analysis
├── code-reviews/                 # PR/MR reviews (structure unchanged)
│   └── <slug>/
│       ├── context.md
│       ├── review.md
│       └── fix-plan.md
├── meta/                         # Generated dashboards
│   ├── dashboard.md              # Dataview-powered status overview
│   └── staleness-report.md       # Generated by staleness script
├── wiki/                         # Session context (claude-obsidian pattern)
│   ├── hot.md                    # ~500-word session cache
│   └── log.md                    # Append-only operation log
├── _templates/                   # Obsidian Templater templates
│   ├── note.md
│   ├── research-paper.md
│   ├── code-review-context.md
│   └── project.md
├── RULES.md                      # Writing guidelines
└── .vault-config.json            # Repo metadata
```

### Key Changes from Current

- **NN- prefixes dropped** — Obsidian sorts alphabetically; use frontmatter `order:` if explicit ordering needed
- **`.repo_paths` → `.vault-config.json`** — structured JSON
- **`wiki/hot.md`** — new, session cache (claude-obsidian pattern)
- **`meta/`** — new, generated dashboards and staleness reports
- **`_templates/`** — new, Obsidian Templater templates
- **No `.png` files** — Excalidraw plugin renders natively

### Slug Derivation

The vault directory name (`<repo-slug>`) is derived from the `repo_id`:
- `repo_id` uses `--` as org/repo separator: `anthropics--claude-code`
- `repo_slug` replaces `--` with `-`: `anthropics-claude-code`
- Local repos (no remote): `local-<dirname>-<hash>` → slug is the same

### .vault-config.json

```json
{
  "repo_id": "anthropics--claude-code",
  "repo_slug": "anthropics-claude-code",
  "repo_root": "/Users/karthik/work1/claude-code",
  "clone_paths": ["/Users/karthik/work1/claude-code"],
  "created": "2026-04-25",
  "version": 3
}
```

## 3. Link Conversion

All links become Obsidian wikilinks:

| Current | Obsidian |
|---|---|
| `[Label](../index.md)` | `[[index\|Label]]` or `[[topic/index\|Label]]` |
| `[Sibling](./02-api.md)` | `[[api]]` |
| `![diagram](./01-auth.png)` | `![[auth.excalidraw]]` |
| `[Cross-ref](../../03-data/index.md)` | `[[data/index\|Data Layer]]` |

### Navigation

**Navigation bars are removed entirely.** Lines matching `> **Navigation:**` and `> **Sub-topics:**` are stripped. Obsidian provides:

- Backlinks panel — automatic "what links here"
- Graph view — visual topic map (replaces Knowledge Map table)
- Outline panel — heading-based navigation within a note

### Knowledge Map

The manual table in `overview.md` becomes a Dataview query:

````markdown
## Knowledge Map

```dataview
TABLE status, file.mtime as "Last Updated"
FROM "notes"
WHERE file.name != "overview"
SORT file.name ASC
```
````

## 5. Hooks

### Current (replaced)

- `SessionStart` → `context-prime` → injects full notes index
- `PostToolUse` (Write/Edit) → re-injects index after note changes

### New

| Hook | Trigger | Action |
|---|---|---|
| `SessionStart` | startup | Read `wiki/hot.md` from active vault — compact session cache |
| `PostToolUse` | Write/Edit | If file is in vault, auto-commit to git |
| `Stop` | session end | Prompt: update `wiki/hot.md` with session summary |

### hooks.json

```json
{
  "hooks": {
    "SessionStart": [{
      "matcher": "startup",
      "hooks": [{
        "type": "command",
        "command": "cat \"$(cat ~/vaults/.active-vault 2>/dev/null)/wiki/hot.md\" 2>/dev/null",
        "timeout": 5
      }]
    }],
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "\"${CLAUDE_PLUGIN_ROOT}/hooks/vault-git-sync\" post-tool-use",
        "timeout": 10
      }]
    }],
    "Stop": [{
      "matcher": "",
      "hooks": [{
        "type": "prompt",
        "prompt": "Update wiki/hot.md in the active vault with a brief summary of what was accomplished this session. Keep it under 500 words. Focus on decisions made, findings, and next steps."
      }]
    }]
  }
}
```

### Active Vault Tracking

`~/vaults/.active-vault` — one-line file containing the path to the currently active vault. Set by `init`/`explore` skills when they resolve which repo the user is in. Avoids re-resolving repo ID on every hook.

### vault-git-sync Script

New bash script at `hooks/vault-git-sync`. On PostToolUse:
1. Reads stdin JSON to get `tool_input.file_path`
2. Checks if path is inside any `~/vaults/*/` directory
3. If yes: `cd <vault> && git add -A && git commit -m "auto: <ISO timestamp>"`
4. If no: exits silently (no output)

### context-prime Script

Deleted. Replaced by the simpler `cat` command in SessionStart hook + `vault-git-sync` script for PostToolUse.

## 6. Plugin Permissions

Plugins cannot auto-grant file access permissions. The `init` skill handles first-run setup:

1. Resolve vault path via `resolve-vault`
2. Check if `~/vaults/` permissions exist in `~/.claude/settings.json`
3. If missing, prompt user and offer to add:

```
Read(~/vaults/**)
Write(~/vaults/**)
Edit(~/vaults/**)
```

Written to `~/.claude/settings.json` (user-level, applies to all projects). One-time setup.

## 7. Migration Script

Converts existing `~/.claude/repo_notes/<repo_id>/` to `~/vaults/<slug>/`.

### CLI Interface

```bash
uv run python -m scripts migrate-to-vault                    # interactive, pick from list
uv run python -m scripts migrate-to-vault --repo-id <id>     # specific repo
uv run python -m scripts migrate-to-vault --all              # migrate everything
uv run python -m scripts migrate-to-vault --dry-run          # preview only
```

### Steps

1. **Discover** — scan `~/.claude/repo_notes/` for all repo dirs, present list
2. **Scaffold** — create `~/vaults/<slug>/` with `.obsidian/`, `_templates/`, `wiki/`, `meta/`
3. **Copy content** — all `.md`, `.excalidraw` files preserving directory structure
4. **Strip NN- prefixes** — rename `01-auth/` → `auth/`, `02-api-endpoints.md` → `api-endpoints.md`. Build a rename map for link fixup
5. **Convert links** — regex pass over all `.md` files:
   - `[Label](relative/path/file.md)` → `[[file-stem|Label]]`
   - `![desc](./file.png)` → `![[file-stem.excalidraw]]`
   - Handle links to `index.md` → `[[folder/index|Folder Name]]`
6. **Remove navigation bars** — strip lines matching `> **Navigation:**` and `> **Sub-topics:**`
7. **Delete PNG files** — don't copy rendered PNGs
8. **Generate `.vault-config.json`** — from old `.repo_paths` + repo ID
9. **Seed `wiki/hot.md`** — initial session cache from `overview.md` summary
10. **Seed `meta/dashboard.md`** — Dataview query for knowledge map
11. **Report** — summary: files moved, links converted, broken links needing manual attention

### Safety

Original `~/.claude/repo_notes/` is never deleted. User removes manually after verifying.

## 12. Out of Scope

- Obsidian plugin development (we use file I/O, not the Obsidian SDK)
- MCP server integration (direct file access is simpler and proven)
- Multi-repo vault merging (one vault per repo is the design)
- Automatic Obsidian plugin installation (user installs Excalidraw, Dataview, Templater manually)
- DragonScale Memory mechanisms (from claude-obsidian — not needed for codebase notes)
